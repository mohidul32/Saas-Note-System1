ğŸ“œ History System - 7 Day Retention

How It Works

1. Automatic Tracking

Every time a note is updated, the save() method automatically creates a history entry with the previous title and content.

2. Storage

History stores: previous title, previous content, timestamp, and the user who made the change.

3. User Access

Users can view all history entries within 7 days and restore any previous version with one click.

4. Automated Cleanup

Celery Beat scheduler runs daily at 2:00 AM to delete history entries older than 7 days.

Implementation Details

# Celery Task (apps/notes/tasks.py) @shared_task def cleanup_old_history(): cutoff = now() - timedelta(days=7) deleted = NoteHistory.objects.filter( changed_at__lt=cutoff ).delete()[0] return deleted # Celery Beat Schedule (config/celery.py) app.conf.beat_schedule = { 'cleanup-old-history': { 'task': 'apps.notes.tasks.cleanup_old_history', 'schedule': crontab(hour=2, minute=0), # Daily at 2 AM } }

Why This Approach?

  • Zero Server Stress: Cleanup runs in background worker, not main server
  • Off-hours Execution: Scheduled at 2 AM when traffic is lowest
  • Asynchronous: Doesn't block user requests
  • Reliable: Celery ensures task completion even if worker restarts
  • Scalable: Can handle millions of records without impacting performance
